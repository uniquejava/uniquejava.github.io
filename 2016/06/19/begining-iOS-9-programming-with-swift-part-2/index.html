使用Custom Cell (UI部分)!

basic style cell里边的thumbnail大小和位置都是固定的, 看起来丑爆了, 接下来我们要做以下几件事:
>

  1. 改变cell的高度

  3. 显示更多restaurant的信息,比如location, type
  4. 设置字体类型和大小
  5. 显示圆角图片(而非矩形)

  1. 把Cell的style从basic改为custom
  2. 把table view的row height 和 cell的row height都改为80(呃..)

Reuse cell导致的bug

滚动table view发现新出现的行随机被打上了勾, 这是因为新出现的行可能复用了缓存中的cell.

解决办法是: 初始化21个false. var restaurantIsVisited = [Bool](count:21, repeatedValue: false)

在打勾的时候(didSelectRowAtIndexPath)将对应的元素设置为true
self.restaurantIsVisited[indexPath.row] = true

在cellForRowAtIndexPath中根据实际情况显示checkmark, 这个方法在滚动table的时候会被逐行调用.

在cellForRowAtIndexPath中重置了cell的accessoryType, 新出现的行不再随机带勾

swipe-to-delete (ch 11)

包含三个步骤:

  1. 打开swipe-to-delete开关
  2. 从data model中删除row
  3. 刷新table view

步骤一

在UITableViewController中加上如下空方法就打开了swift-to-delete的开关(呃..)
func tableView(_:commitEditingStyle:forRowAtIndexPath:)

其中第二个参数UITableViewCellEditingStyle的值有三种.None, .Delete, .Insert

步骤二

在这个方法中判断editingStyle是否为.Delete如果是就从model中删除数据.

步骤三

刷新整个table tableView.reloadData()

或者从table view移除对应的行.
tableView.deleteRowsAtIndexPaths([indexPath],withRowAnimation: .Fade)

最常用的动画效果是.Fade, 可选值有:.Righ, .Left, .Top

swife-for-more (ch 11)

iOS 7 新增的功能, iOS 8对developer开放.

实现tableView(_:editActionsForRowAtIndexPath:)即可, 该方法返回一个Action数组: [UITableViewRowAction]?
注意如果打开了swipe-for-more, swift-to-delete就失效了, 需要自己在此处返回Delele Action.

基本结构

UITableViewRowAction的构造函数如下:
init(style: UITableViewRowActionStyle, title: String?, handler: (UITableViewRowAction, NSIndexPath) -> Void)
