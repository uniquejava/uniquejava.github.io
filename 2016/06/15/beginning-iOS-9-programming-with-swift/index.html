Hello World (chapter 3~4)

唯一不使用Auto Layout的章节.

默认工程

新建工程cmd + shift + n > iOS > single view application

a scene = a view controller + its views
默认生成了一个View Controller Scene其中包括一个View Controller
View Controller中包含
>

  1. View(container view)
  2. Top Layout Guide (这是个标尺, 指向status bar的下边缘或者如果有navigation controller指向nav bar下边缘)
  3. Bottom Layout Guide(指向tab bar的上边缘)

status bar就是最上面显示手机运营商, WIFI, 时间和电池电量的那一部分(高度是固定的20 points)
tab bar是指屏幕最下方可能出现的导航栏.

main.storyboard中的View Controller看起来不是个iPhone, 这是因为iOS8开始引入了size classes, 使用了统一的画布模拟所有的iOS设备. 而在此之前, 你需要给iPhone弄个storyboard, 给iPad弄个storyboard.

在File inspector中反选Use Size Classes就能回到解放前, 此时弹出的对话框让你选择是回退到iPhone模式还是回退到iPad模式. 如果你真的Disable了Size Classes,那么View Controller就看起来像iPhone(或iPad)了.

拖动一个Hello World按钮到View的正中间.

打开ViewController.swift. 加入@IBAction代码

如何绑定按钮的点击事件到View Controller

@IBAction是给interface builder(IB)用的, 标记了@IBAction的方法名在你拖动组件到view controller的时候会出现在候选列表中

Ctrl+拖动按钮到View Controller并选择标记了@IBAction的方法名就能绑定按钮的点击事件.

在Attribute Inspector中设置按钮文字的颜色和按钮的背景色(bgcolor需要向下滚动到view部分才能看到, 这是因为attribute是从子类向上分段设置的)

在Hello World按钮的Connection Inspector中可以看到它发出的事件(Sent Events) 叫Touch Up Inside, 指向了View Controller的showMessage

而在View Controller的Connection Inspector的Received Actions中可以看到它自身的showMessage方法被Hello World按钮的Touch Up Inside所用.

Touch Up Inside是在按钮内部抬起手指的意思, 没有用Touch Down是因为防止误按.(Touch Down挪动手指到其它位置并松开可以避免误按)

使用Auto Layout(chapter 5)

前面章节将Size classes去掉后各类组件是用的固定定位, 别看Hello World按钮在canvas中是居中的,实际上这个按钮是用相对于左上角(0,0)的绝对坐标(x,y)来定位的, 当Cmd + Arrow Left左旋或Cmd + Arrow Right右旋(切换到横屏模式),或者是切换到其它尺寸的iPhone, Hello World按钮就不再居中.

为什么用points而不是pixels

2007年刚推出iPhone的时候屏幕大小为3.5寸 (320x480像素) 那个时候, 1 point = 1 pixel.
后来iPhone4出现了Retina屏 (640*960像素), 1 point = 2 pixes的, 但在设计的时候还是用point来表示, 这样不论是不是Retina屏, 用同一个值就能搞定不同的分辨率.

基准分辨率(单位point)

>
iPhone4/4S: 320x480
iPhone5/5S: 320x568
iPhone6/6S: 375x667

即使屏幕变成640x960或1280x1920, 也只不过是1point变成了2pixel或4pixel, 我们还是用point来做设计,而将单位转换交给iOS系统来处理

公式 button.left = (container.left + 30)

Auto Layout Bar

Each button has its own function:

  1. Stack – Embed views into a stack view. Stack view is a new feature in Xcode 7.
  2. Align – Create alignment constraints, such as aligning the left edges of two views.
  3. Pin – Create spacing constraints, such as defining the width of a UI control.
  4. Issues – Resolve layout issues.

实例一(居中对齐)

比如要居中对齐按钮, 那么先选中按钮, 然后点Align 然后在弹出的popover中选择Vertical center in container and Horizontal center in container 最后点击Add 2 Constraints.

当没有layout issue的时候, constraint line是蓝色的, 此时在canvas中将按钮挪动位置, constraint line会变成橙色, 警告你位置不匹配.

此时可以选中这个按钮, layout bar中点Issues按钮然后选择Update Frames就可以了. 如果要重新给这个按钮添加contraints, 可以点击Clear Constraints

预览

在storyboard左上角点有4个小方格的图标 > Preview(1) > 按住Option键并点击Main.storyboard(Preview), 在打开的界面中点击+号来选择不同尺寸的设备.

实例二 (ctrl + drag label 右0 下20)

比如要给label设定右边距0, 下边距20, 这个边距叫spacing constraints, 在IB中, 可以ctrl drag这个label到它自身, 或是其它参照物来添加constraints.

  1. ctrl drag label向右, 鼠标划过的参照组件会高亮成蓝色, 如果是label本身, 可以设置width, 如果是container view, 可以设定trailing space 和 center vertically
  2. ctrl drag label向左, 设定width或leading space和center vertically
  3. ctrl drag 向上, 设定height或vertical spacing to top 或 center horizontal
  4. ctrl drag 向下, 设定height或vertical spacing to bottom和 center horizontal

注意:

  1. 设定边距的时候不一定要拉到参照物的边缘上. 只要浮到参照物上(高亮成蓝色)就可以.
  2. 无论左右, 拖到自身可以设定width, 拖到superview可以设定center vertically
  3. 无论上下, 拖到自身可以设定height, 拖到superview可以设定center horizontally

实例三(label 上40, 左右居中)

拖个label, ctrl + drag向上, 按住shift, 依次选择vertical spacing to top, center horizontally 和 add contraints 然后在size inspector中修改spacing的值为40, 最后Resolve Issues > Update Frames(快捷键cmd + option + =).

Stack view (chapter 6)

iOS 9引入的新功能. Stack view就是将多个objects分组到一个single object, 起到整体移动或调整大小的作用.
stack view包含的view叫arranged views

  1. stack view
  2. image view
  3. asset catalog
  4. size classes

如何使用?
方式一: 从object lib中拖一个vertical或horizontal stack view到storyboard
方式二: 选中多个ob, 然后点击layout bar上的第一个stack按钮.

关于Assets.xcassets

直接将所有的图片拖到Xcode的Assets.xcassets目录即可.
比如:
>
t-shirt-1.png
t-shirt-1@2x.png
t-shirt-1@3x.png

不带后缀的是给non-Retina屏用的
@2x是给iphone 4/4s/5/5s/6/6s用的
@3x是给iphone plus用的
大小是180x338, 360x676, 540x1015
直接拖到Xcode中的Assets.xcassets目录, Xcode会按照文件名自动分组, 不同的尺寸放在不同的well里.

如果要使用, 直接使用set name(比如t-shirt-1)不需要@后缀,也不需要扩展名, iOS会自动选择合适的图片

实例一(title and subtitle: vertical stack view)

拖一个vertical stack view
拖一个label到stack view里边(stack view会自动调整大小, 包裹住label, 将label size改成50 points)
再拖一个label到stack view里边(当鼠标移到stack view之上stack view会高亮此时释放鼠标)

stackview的属性:

  1. axis: 指定horitonal或vertical
  2. Alignment: 指定arranged views整体靠左,居中还是右对齐
  3. Distribution: 指定arranged views的大小还有位置,默认是Fill - 表示在自己空间允许的范围内尽最大努力包裹住所有的组件. 如果是Fill Equally那么对于vertical stack view会让每个subview的高度相同.
  4. Spacing: 指定arranged views之间的间隔

在label上shift点右键能快速选中某个组件.

在这个例子中, 只需要将Alignment设置成center就可以了.然后将stack view的坐标设置为(183, 46)

实例二(图片: horizontal stack view)

拖一个horizontal stack view
连续向stack view中拖动三个image view, stack view每次都会自动向右增长, 以恰好包裹住所有的image view
设置每个image view的image属性,分别对应三个t-shirt
设置stack view的distribution为Fill Equally, spacing为10
然后将stack view的坐标设置为(20, 142)

到目前为止,我们没有给stack view中的任何subview设置任何的constraints(我们只是设定了stack view的相应属性就让subview正常工作了)

最后一步, 我们要给stack view自身设置constraints.

给stack view自身设置constraints

给包含title的vertical stack view设定约束

  1. vertical spacing to top
  2. centor horizontally
    ctrl drag向上shift选择相应的约束即可.

给包含images的horizontal stack view设定约束:

  1. vertical spacing to 包含title的stack view
  2. leading space to super view margin left
  3. trailing space to superview margin right.

点layout bar上的pin按钮设置上左右为(15.5, 0,0), 注意上左右必须变成红色的实线

因为iPhone portrait模式下, stack view在水平方向上无法容纳三张图片, 此时预览, 会发现图片被挤得细长细长…, 而在垂直方向上, stack view有足够的空间, 而保证了图片的正常高度

怎么让图片在水平挤压的过程中, 适当减小高度, 保持宽高比?
解决办法是: 在stack view上水平ctrl drag拖动自身, 在弹出的popover中选择Aspect Ratio

可见, 即使是让所有的图片保持宽高比都是设定在stack view上的, stack view是所有arranged views的管家.

实例​四(使用Size Classes)

在iPhone landscape模式下. 屏幕的高度不是很够, 希望在此模式下, 去掉title上面的空白, 并减小title和image之间的距离
先前title离上边的距离为26, title离image的距离为15.5, 问题是如何仅在iphone landscape模式下设定这两个值?

iOS8引入了Adaptive Layout, 让你的app能适配特定的device以及特定的orientation, 于是引出了Size Classes

Size Classes是屏幕大小和方向的抽象, 分为​两大类(四小类). 两大类是regular和compact, regular是大/长, compact表示小/短, 再考虑水平方向(width) 和 竖直方向(height), 组合成4种抽象的设备类型.

有点类似于向量.

可以看到左下角是又矮又窄(CC), 右上是又高又宽(RR)

或者

为了确定设备类型, 必须同时指定水平方向上的size class和竖直方向上的size class, 比如iPad 有一个 regular horizontal (width) size class 和一个 regular vertical (height) size class

这里我们要对iPhone的横屏模式做处理, 对应R-C(iphone plus), 和C-C (iphone), 而R-C + C-C = Any-C

点击第一个stack view, 在size inspector中找到常量26所在的位置, 点击
左边的+号, 新增Any width, Compact height并设定值为0
