导航与Segue (ch 13)

scene就是一屏, segue就是屏与屏之间的transition
In a storyboard, a scene usually refers to the on-screen content (e.g. a view controller). Segues sit between two scenes and represent the transition from one scene to another. Push and Modal are two common types of transition.

新建navigation controller

给view controller添加navagation bar的方法:

  1. 选中view controller, 然后Editor > Embedded in > Navigation controller
  2. 在当前的view controller顶部多出一个叫Navigation Item的东西, 这个就是navigation bar
  3. 可以选中它, 并在Attribute Inspector修改它的title属性

新建detail controller

  1. 拖一个view controller到storyboard
  2. 拖一个image view到view controller并设置image为全屏
  3. 修改image view的mode,从Scale to Fill改为Aspect Fill.

  4. 新建RestaurantDetailViewController继承自UIViewController并设置为view controller的custom class.

设置image全屏的方法

拉动image的左上和右下让它与superview边缘对齐, 点击pin按钮然后点击上右下左那4条bar让它们变成红色的实线最后Add 4 constraints即可

导航: 从list view到detail view

Ctrl拖动prototype cell到新的view controller, 在弹出的菜单中选择Selection Segue - Show

Segue的类型

4种类型的segues:
>

  1. show 新页面push到当前页面之上, 新页面会显示后退按钮
  2. show detail 新页面替换当前页面, 不会显示后退按钮
  3. present modally 从底部滑出并占满整屏(类似calendar点击+)
  4. present as popover (类似tooltip)

给segue命名

两个view controller之间很可能会有多个segue, 选择某个segue, 打开attributes inspector, 在identifier一栏给segue指定唯一标识符(e.g.: showRestaurantDetail)

从list view传参到detail view

prepareForSegue(:segue), UITableViewController的模板代码中已包含了这个方法, 只是默认注释掉了.

设定header(图片部分)

步骤如下:

  1. 清除当前detail view中的全部内容(上节中的全屏image)
  2. 拖一个table view到view controller并设置为全屏(上边界要拉到nav bar的上边缘)
  3. 选中table view, 设定prototype cells=1(cell默认为custom类型)
  4. 选中Cell,设置identifier=Cell, row height=36(默认值)
  5. 拖一个image view到table view做为它的header -image view是table view的第一个child, cell是table view的第二个child, Image View和Cell平级) - 最简单的方法是直接把image view拖动到document outline对应的位置.
  6. 设置image view的height=300 (bounds为0,64,600,300)
  7. 建立image view和outlet的连接(在view controller上右键并circle)

UIImageView的拉伸模式

UIImageView继承自UIView, 它有一个mode属性用来决定view如何包裹它的内容.

三种模式(page 236)
>
Scale to Fill(默认值, 填充View, 图片可能会被拉伸变形)
Aspect Fit(等比缩放, 但是图片的上下或左右可能出现空白)
Aspect Fill(等比缩放, 但是图片的某边可能超出边界)

最佳组合: Aspect Fill并且和With Clip Subview一起使用, 这个选项能裁掉多出的部分

Custom Cell(UI部分)

参见part 2 custom cell部分

  1. Label(name=Field, font style=Medium)
  2. Label(name=Value)
  3. 将这两个label装进horizontal stack view
  4. 设定hsv自身的constraint(center vertically, leading, trailing=0), 方法是先ctrl drag hsv向右, 再ctrl drag hsv向左.(注意后续自适应cell大小的章节我们还需要设置它的top=0, bottom=0)

  5. 设定trailing spacing contraint的constant为0, leading的没做要求.

  6. 设定Field label的Content Hugging Priority把Horizontal从251改成261, 这样Field label比value有更高优先级享有intrinsic size的权利(Field的宽度将自动调整到刚好容纳自身的大小, 这个大小叫intrinsic size)

  7. 设定Field label是Value label宽度的50%, 方法是ctrl drag Field到Value之上并选择Equal width, 然后将multiplier从1改为0.5

什么是Dynamic Type

如果Cell能自适应大小(见上一节), 并且cell中label的字体是text style类型(比如Headline),而不是固定大小的font type, 这时这个label的字体大小就会随用户的设置动态改变.

用户设置字体大小的步骤: Settings > General > Accessibility > Larger Text or Settings > Display & Brightness > Text Size
