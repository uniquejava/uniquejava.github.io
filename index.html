为什么用points而不是pixels

2007年刚推出iPhone的时候屏幕大小为3.5寸 (320x480像素) 那个时候, 1 point = 1 pixel.
后来iPhone4出现了Retina屏 (640*960像素), 1 point = 2 pixes的, 但在设计的时候还是用point来表示, 这样不论是不是Retina屏, 用同一个值就能搞定不同的分辨率.

基准分辨率(单位point)

>
iPhone4/4S: 320x480
iPhone5/5S: 320x568
iPhone6/6S: 375x667

即使屏幕变成640x960或1280x1920, 也只不过是1point变成了2pixel或4pixel, 我们还是用point来做设计,而将单位转换交给iOS系统来处理

实例二 (ctrl + drag label 右0 下20)

比如要给label设定右边距0, 下边距20, 这个边距叫spacing constraints, 在IB中, 可以ctrl drag这个label到它自身, 或是其它参照物来添加constraints.

  1. ctrl drag label向右, 鼠标划过的参照组件会高亮成蓝色, 如果是label本身, 可以设置width, 如果是container view, 可以设定trailing space 和 center vertically
  2. ctrl drag label向左, 设定width或leading space和center vertically
  3. ctrl drag 向上, 设定height或vertical spacing to top 或 center horizontal
  4. ctrl drag 向下, 设定height或vertical spacing to bottom和 center horizontal

注意:

  1. 设定边距的时候不一定要拉到参照物的边缘上. 只要浮到参照物上(高亮成蓝色)就可以.
  2. 无论左右, 拖到自身可以设定width, 拖到superview可以设定center vertically
  3. 无论上下, 拖到自身可以设定height, 拖到superview可以设定center horizontally

stackview的属性:

  1. axis: 指定horitonal或vertical
  2. Alignment: 指定arranged views整体靠左,居中还是右对齐
  3. Distribution: 指定arranged views的大小还有位置,默认是Fill - 表示在自己空间允许的范围内尽最大努力包裹住所有的组件. 如果是Fill Equally那么对于vertical stack view会让每个subview的高度相同.
  4. Spacing: 指定arranged views之间的间隔

在label上shift点右键能快速选中某个组件.

在这个例子中, 只需要将Alignment设置成center就可以了.然后将stack view的坐标设置为(183, 46)

给stack view自身设置constraints

给包含title的vertical stack view设定约束

  1. vertical spacing to top
  2. centor horizontally
    ctrl drag向上shift选择相应的约束即可.

给包含images的horizontal stack view设定约束:

  1. vertical spacing to 包含title的stack view
  2. leading space to super view margin left
  3. trailing space to superview margin right.

点layout bar上的pin按钮设置上左右为(15.5, 0,0), 注意上左右必须变成红色的实线

因为iPhone portrait模式下, stack view在水平方向上无法容纳三张图片, 此时预览, 会发现图片被挤得细长细长…, 而在垂直方向上, stack view有足够的空间, 而保证了图片的正常高度

怎么让图片在水平挤压的过程中, 适当减小高度, 保持宽高比?
解决办法是: 在stack view上水平ctrl drag拖动自身, 在弹出的popover中选择Aspect Ratio

可见, 即使是让所有的图片保持宽高比都是设定在stack view上的, stack view是所有arranged views的管家.

先前title离上边的距离为26, title离image的距离为15.5, 问题是如何仅在iphone landscape模式下设定这两个值?

iOS8引入了Adaptive Layout, 让你的app能适配特定的device以及特定的orientation, 于是引出了Size Classes

Size Classes是屏幕大小和方向的抽象, 分为​两大类(四小类). 两大类是regular和compact, regular是大/长, compact表示小/短, 再考虑水平方向(width) 和 竖直方向(height), 组合成4种抽象的设备类型.

有点类似于向量.

可以看到左下角是又矮又窄(CC), 右上是又高又宽(RR)

或者

为了确定设备类型, 必须同时指定水平方向上的size class和竖直方向上的size class, 比如iPad 有一个 regular horizontal (width) size class 和一个 regular vertical (height) size class

这里我们要对iPhone的横屏模式做处理, 对应R-C(iphone plus), 和C-C (iphone), 而R-C + C-C = Any-C
